NGN virtual accounts let your customers pay you by bank transfer. Fincra generates a unique account number — your customer transfers funds into it, and your Fincra NGN wallet is credited automatically.

## Permanent virtual accounts

Permanent accounts are static and reusable — the account number never changes. Assign one per customer for recurring payments or wallet funding.

**Requirements:**
- BVN is required for all NGN permanent virtual account requests
- NGN virtual account requests are approved instantly

### Supported banks

| Bank | Channel value | Status |
|---|---|---|
| Globus Bank | `globus` | Active |
| Wema Bank | `wema` | Active |
| Sterling Bank | `sterling` | Active |
| Guaranty Trust Bank | `habari` | Active |

If no `channel` is specified, `wema` is used by default.

### Create a permanent virtual account

**Endpoint:** `POST /profile/virtual-accounts/requests`

**Headers:**

| Header | Description |
|---|---|
| `api-key` | Your API secret key |
| `Content-Type` | `application/json` |

<Tabs>
  <Tab title="Individual">
    ```bash
    curl --request POST \
      --url https://api.fincra.com/profile/virtual-accounts/requests \
      --header 'api-key: <YOUR_SECRET_KEY>' \
      --header 'Content-Type: application/json' \
      --data '{
        "currency": "NGN",
        "accountType": "individual",
        "KYCInformation": {
          "firstName": "Amara",
          "lastName": "Okonkwo",
          "email": "amara@example.com",
          "bvn": "12345678901"
        }
      }'
    ```

    | Field | Type | Required | Description |
    |---|---|---|---|
    | `currency` | string | Yes | Must be `NGN`. |
    | `accountType` | string | Yes | Must be `individual`. |
    | `KYCInformation.firstName` | string | Yes | Customer's first name. |
    | `KYCInformation.lastName` | string | Yes | Customer's last name. |
    | `KYCInformation.email` | string | No | Customer's email address. |
    | `KYCInformation.bvn` | string | Yes | Customer's BVN. |
    | `channel` | string | No | Bank to use. Defaults to `wema`. See [supported banks](#supported-banks). |
    | `merchantReference` | string | No | Your unique reference for this account. |
  </Tab>
  <Tab title="Corporate">
    The BVN provided for a corporate account must match a director or shareholder on the CAC record for the corporate entity.

    ```bash
    curl --request POST \
      --url https://api.fincra.com/profile/virtual-accounts/requests \
      --header 'api-key: <YOUR_SECRET_KEY>' \
      --header 'Content-Type: application/json' \
      --data '{
        "currency": "NGN",
        "accountType": "corporate",
        "KYCInformation": {
          "bvn": "90909090909",
          "bvnName": "Jane Foster",
          "businessName": "Jane Corp"
        },
        "channel": "wema"
      }'
    ```

    | Field | Type | Required | Description |
    |---|---|---|---|
    | `currency` | string | Yes | Must be `NGN`. |
    | `accountType` | string | Yes | Must be `corporate`. |
    | `KYCInformation.bvn` | string | Yes | BVN of a director or shareholder. |
    | `KYCInformation.bvnName` | string | Yes | Name as it appears on the BVN. Must match the account name. |
    | `KYCInformation.businessName` | string | Yes | Registered business name. |
    | `KYCInformation.email` | string | No | Business email address. |
    | `channel` | string | No | Bank to use. Defaults to `wema`. |
  </Tab>
</Tabs>

**Response:**

```json
{
  "success": true,
  "message": "Virtual account created successfully",
  "data": {
    "status": "approved",
    "isActive": true,
    "isPermanent": true,
    "accountInformation": {
      "accountNumber": "3992219528",
      "accountName": "Amara Okonkwo",
      "bankName": "GLOBUS BANK",
      "bankCode": "103"
    },
    "KYCInformation": {
      "firstName": "Amara",
      "lastName": "Okonkwo",
      "email": "amara@example.com"
    },
    "currency": "NGN",
    "accountType": "individual",
    "virtualAccountType": "additional",
    "_id": "65xxxxxxxxxxxxxxxxxxx",
    "business": "67xxxxxxxxxxxxxxxxxxx",
    "createdAt": "2026-02-19T09:00:00.000Z",
    "updatedAt": "2026-02-19T09:00:00.000Z"
  }
}
```

| Field | Description |
|---|---|
| `data._id` | Unique identifier for the virtual account. Use this to retrieve or reference the account. |
| `data.accountInformation.accountNumber` | The account number to share with your customer. |
| `data.virtualAccountType` | `main` for the first account created, `additional` for all subsequent accounts. |
| `data.isPermanent` | `true` for permanent accounts. |

### Receive the webhook

When a customer transfers funds into the virtual account, Fincra sends a `charge.successful` webhook to your configured webhook URL.

```json
{
  "event": "charge.successful",
  "data": {
    "reference": "fcr-bt-xxxxxxxxxxxx",
    "amount": 10000,
    "amountReceived": 10000,
    "amountToSettle": 9750,
    "currency": "NGN",
    "status": "success",
    "type": "bank_transfer",
    "virtualAccount": {
      "accountNumber": "3992219528",
      "accountName": "Amara Okonkwo",
      "bankName": "GLOBUS BANK",
      "payerAccountNumber": "0123456789",
      "payerAccountName": "Sender Name",
      "payerBankName": "First Bank"
    }
  }
}
```

Use the `virtualAccount.accountNumber` field to match the incoming payment to the correct customer in your system.

For full webhook setup, see [Webhooks](/webhooks/overview).

## Temporary virtual accounts

Temporary accounts expire at a time you set. They are ideal for one-time payment scenarios — generate a unique account number for a transaction, the customer pays into it, and the account expires after use.

**Temporary virtual accounts can only be created via the API.**

### Simulate a payment in sandbox

To test VA payments in the sandbox environment without sending real money, call the sandbox webhook simulation endpoint:

```bash
curl --request POST \
  --url https://sandboxapi.fincra.com/collections/webhooks/wema/callback \
  --header 'api-key: <YOUR_SANDBOX_SECRET_KEY>' \
  --header 'Content-Type: application/json' \
  --data '{
    "accountNumber": "<VIRTUAL_ACCOUNT_NUMBER>",
    "amount": 10000
  }'
```

Your Fincra sandbox wallet will be credited instantly and a `charge.successful` webhook will fire.

## Retrieve virtual accounts

**Get a single virtual account by ID:**

```bash
curl --request GET \
  --url https://api.fincra.com/profile/virtual-accounts/<virtual_account_id> \
  --header 'api-key: <YOUR_SECRET_KEY>'
```

**List all virtual accounts:**

```bash
curl --request GET \
  --url https://api.fincra.com/profile/virtual-accounts?currency=NGN \
  --header 'api-key: <YOUR_SECRET_KEY>'
```
