## 1. Initiate a charge

**Endpoint:** `POST /checkout-core/charges`

**Headers:**

| Header         | Description         |
| -------------- | ------------------- |
| `api-key`      | Your API secret key |
| `Content-Type` | `application/json`  |

**Base request fields:**

| Field            | Type   | Required | Description                                                                                  |
| ---------------- | ------ | -------- | -------------------------------------------------------------------------------------------- |
| `type`           | string | Yes      | Payment method. See [supported values](/payments/direct-charge#supported-payment-methods).   |
| `amount`         | number | Yes      | The amount to charge.                                                                        |
| `currency`       | string | Yes      | The currency to charge in.                                                                   |
| `reference`      | string | No       | Your unique reference for this transaction. Auto-generated if not provided.                  |
| `customer`       | object | Yes      | Customer details.                                                                            |
| `customer.name`  | string | Yes      | Customer's full name.                                                                        |
| `customer.email` | string | Yes      | Customer's email address.                                                                    |

Additional fields are required depending on the payment method. See the examples below.

## 2. Authorize a charge

Some payment methods require the customer to authenticate the transaction before it completes.

**Card — OTP**

If the response contains `"mode": "otp"`, prompt your customer to enter the OTP sent to their phone or email, then submit it to the authorize endpoint.

<Info>Authorize endpoint and payload — coming soon.</Info>

**Card — 3D Secure**

If the response contains `"mode": "redirect"`, redirect the customer to the URL in the `redirect` field to complete 3DS authentication. After authentication, the customer is returned to your site.

**Mobile Money**

No API call required — the customer receives a prompt on their mobile device (e.g., M-Pesa, MTN, Airtel) and approves the payment directly. Poll the verify endpoint to check when the payment is confirmed.

## 3. Verify the charge

Always verify the charge server-side before delivering value. Use any of the three endpoints:

<Tabs>
  <Tab title="By Fincra reference">
    ```bash
    curl --request GET \
      --url https://api.fincra.com/checkout-core/charges/reference/order-001 \
      --header 'api-key: <YOUR_SECRET_KEY>'
    ```
  </Tab>
  <Tab title="By merchant reference">
    ```bash
    curl --request GET \
      --url https://api.fincra.com/checkout-core/charges/merchant-reference/order-001 \
      --header 'api-key: <YOUR_SECRET_KEY>'
    ```
  </Tab>
  <Tab title="By charge ID">
    Auto-validates if the charge is still pending or processing. Use `?statusCheck=true` to trigger a live status check.

    ```bash
    curl --request GET \
      --url 'https://api.fincra.com/checkout-core/charges/<id>/verify?statusCheck=true' \
      --header 'api-key: <YOUR_SECRET_KEY>'
    ```
  </Tab>
</Tabs>

Check the `status` field before delivering your product or service:

| Status       | Meaning                                                     |
| ------------ | ----------------------------------------------------------- |
| `success`    | Payment confirmed. Safe to deliver.                         |
| `pending`    | Payment not yet completed. Continue polling.                |
| `processing` | Payment is being processed. Continue polling.               |
| `failed`     | Payment failed or was abandoned.                            |

## 4. Handle the webhook

Fincra sends a `charge.successful` event to your configured webhook URL when the payment is confirmed.

```json
{
  "event": "charge.successful",
  "data": {
    "reference": "fcr-p-2fe90fa19d",
    "merchantReference": "order-001",
    "amount": 100,
    "amountReceived": 100,
    "amountToSettle": 97,
    "currency": "NGN",
    "fee": 3,
    "status": "success",
    "type": "bank_transfer",
    "feeBearer": "business",
    "customer": {
      "name": "Amara Okonkwo",
      "email": "amara@example.com"
    }
  }
}
```

For full webhook setup and signature verification, see [Webhooks](/webhooks/overview).
