Different currencies use different numbers of decimal places. Sending an amount with the wrong precision will cause payout failures. Use this table to ensure your amounts are formatted correctly for each currency.

## Precision table

Fincra follows the ISO 4217 standard for currency representation.

| Currency | Code | Decimal places | Example |
|---|---|---|---|
| Nigerian Naira | NGN | 2 | `1000.50` |
| Ghanaian Cedi | GHS | 2 | `500.00` |
| Kenyan Shilling | KES | 2 | `25000.00` |
| Ugandan Shilling | UGX | 2 | `100000.00` |
| Zambian Kwacha | ZMW | 2 | `750.00` |
| Tanzanian Shilling | TZS | 2 | `50000.00` |
| Central African CFA Franc | XAF | 2 | `10000.00` |
| West African CFA Franc | XOF | 2 | `10000.00` |
| South African Rand | ZAR | 2 | `1500.00` |
| Egyptian Pound | EGP | 2 | `3000.00` |
| US Dollar | USD | 2 | `100.50` |
| British Pound | GBP | 2 | `250.75` |
| Euro | EUR | 2 | `500.25` |

<Note>
  All payout currencies currently use **2 decimal places**. This table will be updated if precision rules change for specific currencies. Always format your amounts with exactly 2 decimal places to avoid validation errors.
</Note>

## How to use this

When building your integration, format the `amount` field in your payout request to match the decimal precision for the destination currency:

```json
{
    "amount": 1000.50,
    "destinationCurrency": "NGN"
}
```

Sending an amount with more decimal places than the currency supports (e.g., `1000.505` for NGN) may result in a validation error or unexpected rounding.
